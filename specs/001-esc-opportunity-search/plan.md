# Implementation Plan: ESC Opportunity Search

**Branch**: `001-esc-opportunity-search` | **Date**: 2026-03-30 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-esc-opportunity-search/spec.md`

## Summary

Build a Python MCP server and data ingestion pipeline that enables conversational search over European Solidarity Corps volunteering opportunities. The ingestion script fetches all NL-eligible opportunities from the ESC portal's Elasticsearch API, scrapes deadlines from individual pages, and stores everything in a Databricks Delta table with a Vector Search index. The MCP server exposes four tools (search, filter, details, stats) that Claude can call from Alex's desktop app.

## Technical Context

**Language/Version**: Python 3.12+
**Primary Dependencies**: `mcp[cli]` (MCP server SDK), `databricks-vectorsearch` (vector search client), `databricks-sdk` (Delta table operations), `httpx` (async HTTP for API + scraping)
**Storage**: Databricks Delta table + Vector Search index (Free Edition)
**Testing**: pytest with recorded fixtures for ESC API responses, real Databricks calls for integration tests
**Target Platform**: Linux (Proxmox LXC container), stdio MCP transport
**Project Type**: MCP server + data pipeline
**Performance Goals**: Search results within 5 seconds, daily ingestion completes within 30 minutes
**Constraints**: Single user (Alex), ~500–3,000 opportunities, Databricks Free Edition quotas
**Scale/Scope**: ~3,000 records, 1 user, 4 MCP tools, 1 daily ingestion job
